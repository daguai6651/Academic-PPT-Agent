## 一、项目概述

### 1.1 项目背景

现有 AI PPT 生成工具的核心痛点：

1. **全局修改，无法局部微调** — 用户说"把这页某个图标换掉"，AI 把整页甚至整份 PPT 重新生成，越改越差
2. **生成后难以修改** — 导出为 PPTX 后，AI 生成的内容不支持继续编辑
3. **缺乏学术场景优化** — 现有工具针对商业汇报，不适配学术论文的结构
4. **PDF 解析质量差** — 传统 PDF 解析工具无法正确处理双栏排版、公式、表格

### 1.2 项目目标

打造一个 **"论文 → 学术 PPT → 交互式局部修改"** 的智能体系统：

- ✅ **论文自动解析** — 支持 PDF 输入，自动提取结构化内容
- ✅ **学术模板生成** — 自动适配计算机论文汇报的标准结构（8-12 页）
- ✅ **局部交互修改** — 点击选中元素 → AI 生成修改方案 → accept/reject（类似 Cursor）
- ✅ **多格式导出** — HTML 网页演示 / PPTX / PDF
- ✅ **公式/架构图渲染** — LaTeX 公式、论文原图、实验表格

### 1.3 目标用户

计算机专业研究生/本科生（学术汇报、组会、毕业答辩）

```

```
输入（PDF 文件）

#### LaTeX → HTML 转换详情

`latex_to_html()` 函数处理的结构：

| LaTeX 结构 | HTML 实现 | 示例 |
|-----------|----------|------|
| `\frac{分子}{分母}` | inline-flex 上下结构 + 横线 | QK^T / √d_k |
| `\sqrt{内容}` | √ 符号 + 上边框 | √d_k |
| `^{上标}` | `<sup>` 标签 | K^T |
| `_{下标}` | `<sub>` 标签 | d_k |
| `\text{文本}` | 纯文本 | Attention |
| `\mathrm{}`、`\mathbf{}` | 粗体/斜体标签 | **W** |
| 数学符号（sin/cos/α/β/Σ 等） | HTML 字符 | sin, α, Σ |

#### 图片嵌入方案

- **方式：** Base64 内嵌（`data:image/png;base64,...`）
- **优势：** JSON 文件自包含，不需要额外管理图片目录
- **手动映射：** 当前手动指定图片到页面的对应关系，后续改为自动定位
